📚 Tuning Guide & Calculators
0. PID Primer — What Each Term Does
  • P (Kp): reacts to current error. Higher = faster/oscillatory. Leaves offset.
  • I (Ti): eliminates offset via accumulated error. Smaller Ti = more aggressive.
  • D (Td): reacts to rate of PV. Damps overshoot. Amplifies noise.

DCS controller forms

FormEquationWhere
ISA (ideal)Kp·[e + (1/Ti)∫e + Td·de/dt]Honeywell, ABB, this tool
ParallelKp·e + Ki·∫e + Kd·de/dtAllen-Bradley Logix
SeriesKp·(1+1/(Ti·s))·(1+Td·s)Foxboro, legacy
⚠ Same Kp/Ti/Td give different responses in different forms. Always confirm before copying tunes between DCSs.
1. Open-Loop Step Test

Every tuning method needs three numbers: Kp, τ, θ. Get them from a step test.

Kp = ΔPV / ΔOP
θ = time from step until PV moves
τ = time from end of θ until PV reaches 63.2% of ΔPV

v2.24.7+ automation: Auto Sequence Identify + Tune mode automates this with 8 OP steps (2↑ 2↓ 2↓ 2↑) and fits FOPDT model automatically.

4. Lambda Tuning
Kc = τ/[Kp·(λ+θ)], Ti = τ

λ is the closed-loop time constant desired. Larger λ = slower, more robust.

⭐ 5. SIMC (Skogestad) — ACTIVE METHOD
Kc = τ/[Kp·(λ+θ)]
SIMC classic: Ti = min(τ, 4·(λ+θ))
Improved SIMC (v2.24.5+): If τ/θ > 5 → Ti = 4·(λ+θ) WITHOUT τ cap
Td = θ/2 (only for PID)

Auto-Tune uses SIMC: Conservative λ=8θ, Balanced λ=3θ, Aggressive λ=θ. Slider modulates λ ×0.5 to ×2.

⭐ 6. Damping Estimation from Overshoot (v2.24.7+)

Post-verify recommendation uses second-order underdamped analysis:

ζ = −ln(OS/100) / √(π² + ln²(OS/100))

Where OS is measured overshoot %. From ζ derive corrected λ:

λ_new = λ_old · (ζ_target / ζ_observed)²

Target ζ = 0.7 (well-damped, ~5% overshoot). Then SIMC applied with λ_new.

7. Cohen-Coon

Best when θ/τ > 0.3. Let r = θ/τ.

Kc = (1/Kp)·(τ/θ)·(4/3+r/4)
8. IMC (Internal Model Control)
Kc = (1/Kp)·(τ+θ/2)/(τc+θ/2), Ti = τ+θ/2, Td = τ·θ/(2τ+θ)
9. Manual Trial & Error
  1. Start Ti = 9999, Td = 0, Kp = 0.5.
  2. Step SP: sluggish → double Kp; oscillates → halve Kp.
  3. Reduce Ti (start ~3·θ) until offset vanishes without oscillation.
  4. Clean PV + overshoot → add Td ≈ θ/2.

10. Relay Autotune (Åström-Hägglund)
Ku = 4·h / (π·a)

Safer than closed-loop Z-N. Built into Honeywell LOOPTUNE, Emerson InSight, ABB AdaptivePID.

11. Mining-Specific Heuristics

Flow / pressure

  • PI only. Lambda λ=τ. Kc 0.3–0.8, Ti 5–20 s.

Level (surge tanks)

  • Ti puede ser 3-8× τ (Improved SIMC lo permite). Tight: λ≈θ. Surge: λ=5τ+.

Density / pulp

  • Slow, very noisy. Kc 0.2–0.5, Ti 60–180 s, no Td.

SAG mill load

  • Kc 0.1–0.3, Ti 120–300 s.

Temperature

  • Cohen-Coon or SIMC λ=θ. Include Td.

Sanity checks

  • Kc·Kp ≈ 0.3–1.5 for most PI loops
  • Ti ≈ τ (Lambda) or 2–4·θ (Z-N)
  • Td ≤ θ always
  • Overshoot < 15%, Gm ≥ 6 dB, Pm ≥ 45°
  • Target damping ζ ≈ 0.7 for balanced response
